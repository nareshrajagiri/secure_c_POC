RELEVANT SECURE-C RULES:


Rule ID: DCL31-C

Category: DCL

Description:
All identifiers must be declared before their first use to ensure proper type checking, avoid implicit declarations, and prevent undefined behavior.

Fix:
Declare all variables and functions before their first usage, typically at the beginning of a block or in appropriate header files.



Rule ID: EXP45-C

Category: EXP

Description:
Do not perform assignment operations within selection statements such as if, while, or for conditions. Assignments inside selection statements can lead to readability issues, introduce subtle bugs, and make the code harder to maintain and debug. Instead, perform all assignments separately before the selection statement.

Fix:
Assign values to variables before the selection statements and use only expressions in the conditions; for example, replace 'if ((x = func()) != 0)' with 'x = func(); if (x != 0)'.



Rule ID: ERR33-C

Category: ERR

Description:
Detect and properly handle error indications returned by standard library functions to avoid undefined behavior or security vulnerabilities caused by ignoring errors.

Fix:
Check return values of standard library functions, handle errors immediately by implementing recovery, cleanup, or safe fallback procedures suitable for STM32 embedded environments.



Rule ID: DCL36-C

Category: DCL

Description:
Do not declare an identifier with conflicting linkage classifications, as this leads to undefined behavior and linkage errors. Ensure all declarations of the same identifier within and across translation units have consistent linkage specifications.

Fix:
Ensure that all declarations of an identifier use the same linkage (e.g., all extern or all static) across all files, avoiding mixing linkage types for the same name.



Rule ID: DCL40-C

Category: DCL

Description:
Do not create incompatible declarations of the same function or object. Incompatible declarations can lead to undefined behavior due to mismatched types, calling conventions, or storage durations. Ensure all declarations and definitions consistently use the same type, qualifier, and linkage.

Fix:
Ensure all declarations and definitions of a function or object use exactly the same type, qualifiers, and linkage specifications. In STM32 projects, declare functions in header files with matching prototypes and include these headers consistently across source files.

RELEVANT SECURE-C RULES:


Rule ID: DCL31-C

Category: DCL

Description:
All identifiers must be declared before their first use to ensure proper type checking, avoid implicit declarations, and prevent undefined behavior.

Fix:
Declare all variables and functions before their first usage, typically at the beginning of a block or in appropriate header files.



Rule ID: DCL40-C

Category: DCL

Description:
Do not create incompatible declarations of the same function or object. Incompatible declarations can lead to undefined behavior due to mismatched types, calling conventions, or storage durations. Ensure all declarations and definitions consistently use the same type, qualifier, and linkage.

Fix:
Ensure all declarations and definitions of a function or object use exactly the same type, qualifiers, and linkage specifications. In STM32 projects, declare functions in header files with matching prototypes and include these headers consistently across source files.



Rule ID: MSC40-C

Category: MSC

Description:
Do not violate constraints imposed by the language or the implementation, such as accessing memory out of bounds, using uninitialized variables, or exceeding hardware limitations. Such violations can lead to undefined behavior, system crashes, or security vulnerabilities.

Fix:
Validate all inputs, initialize variables before use, and enforce memory boundaries using static or dynamic checks appropriate for embedded environments.



Rule ID: MSC38-C

Category: MSC

Description:
Do not treat a predefined identifier as an object if it might only be defined as a macro or a compilerIntrinsic, to avoid undefined behavior or conflicts.

Fix:
Avoid declaring variables or objects with names matching predefined identifiers; use distinct names or enclosed preprocessing directives to differentiate.



Rule ID: MSC37-C

Category: MSC

Description:
Ensure that control never reaches the end of a non-void function without an explicit return statement, to prevent undefined behavior and ensure deterministic function outcomes.

Fix:
Add an explicit return statement with an appropriate value before the end of all non-void functions.
